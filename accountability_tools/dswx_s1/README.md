# DSWx-S1 Accountability

To run DSWx-S1 accountability, run the following 4 scripts:

## Setup

Before running the scripts, you must first prepare the environment:

```shell
python -m venv venv
source venv/bin/activate

pip install -r requirements.txt
```

You should also confirm the sqlite database in this repo is the latest MGRS tile database for the DSWx-S1 product. Check
with a PGE/PCM developer for this.

If a new database is needed, copy it to the directory, update the `MGRS_TILE_DB` constant in `missing_rtcs_to_tile_sets.py`
to the new database's filename, and check these changes into the repo.

## 1 - survey.py

survey.py pulls and dedupes all the RTC and DSWx-S1 products from CMR over a given time range. The suggested start date is `2024-08-21T00:11:56Z`
(the earliest sensing time of DSWx-S1) and the suggested end date is the last second of the month prior to running (ie, 
running in mid March 2026, the suggested end date would be `2026-02-28T23:59:59Z`). 

**NOTE:** This script can take a long time to run, so it is suggested to run on a remote machine with nohup:

```shell
nohup python survey.py -s 2024-08-21T00:11:56Z -e 2026-02-28T23:59:59Z 2>&1 > survey.log &
```

This script will produce 2 JSON output files: `rtc_products.json` and `dswx_products.json`

These files contain the relevant results from CMR:

## 3 - missing_rtcs_to_tile_sets.py

missing_rtcs_to_tile_sets.py maps the list of missing RTC inputs created by accountability.py to MGRS tile set IDs, while
also filtering out tile sets where the `land_ocean_flag` is `water`, because these should never be triggered.

**NOTE:** This script can take a long time to run, so it is suggested to run on a remote machine with nohup:

```shell
nohup python missing_rtcs_to_tile_sets.py 2>&1 > mgrs_map.log &
```

This script will produce a single binary output for the mappings of tile set IDs to lists of corresponding missing RTCs: 
`missing_rtcs_to_tile_sets.pickle`

and sensor.
